Why LintScout?

Linters exist for a reason. Every ignore directive is a decision to bypass a safety net. LintScout makes those decisions visible.

⚡

Blazing Fast

Built in Rust using the same filesystem engine as ripgrep. Scans 10,000+ files in under 100ms.

🛠

22 Built-in Scouts

Covers ESLint, TypeScript, Pylint, Flake8, golangci-lint, Clang-Tidy, RuboCop, ShellCheck, and 14 more out of the box.

🔧

Fully Configurable

YAML config for excludes, thresholds, scout selection, and custom regex-based rules. CLI flags override everything.

🚀

CI/CD Ready

Exit codes, pass thresholds, and quiet mode make it trivial to add to GitHub Actions, GitLab CI, or any pipeline.

📊

JSON Output

Machine-readable output for dashboards, trend tracking, and integration with your existing toolchain.

Get started in 30 seconds

1

Install

cargo install lintscout
2

Run

lintscout .
3

Add to CI (optional)

lintscout --pass-threshold 0 --format json .

Flexible by design

Filter scouts, set thresholds, export JSON, define custom rules.

Filter by language

lintscout --scouts pylint,flake8,mypy,bandit src/

JSON for dashboards

lintscout --format json | jq '.stats.findings_count'

CI gate

lintscout --pass-threshold 0 --quiet .

Custom config

lintscout --config .lintscout.yml src/

Example .lintscout.yml

settings:
  exclude: [node_modules, vendor, dist]
  pass_threshold: 10
  output: text
  disable:
    scouts: [jshint, prettier]

scouts:
  - name: todo-fixme
    extensions: [js, ts, py, go]
    rules:
      - id: todo
        pattern: "TODO|FIXME|HACK"
